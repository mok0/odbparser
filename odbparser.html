OdbParser -- a Python extension module to read O files

OdbParser is a module for the Python programmer who would like to extract information from O files. The use of the OdbParser module is probably best illustrated by an example. First, we import the module, and open up an O data file. This file could be both binary and formatted, and it could contain any number of datablocks. For this example, we choose a binary file as written by O when it exits: 
>>> import odbparser
>>> db = odbparser.get("binary.O")
Reading binary O file
Now, after reading the file, db is an ordinary Python dictionary, the keys are names of datablocks, and the content of each entry is whatever the type of that data is. For example, let us look at a few well-known O datablocks: 
>>> type (db[".gs_real"])

>>> print db[".gs_real"]
[  3.18265676e-01   1.38070002e+01   3.88569984e+01   2.27169991e+01
   6.26805648e-02  -5.41465655e-02   3.07297319e-01   0.00000000e+00
  -1.21590950e-01  -2.92898685e-01  -2.68076919e-02   0.00000000e+00
   2.87366927e-01  -1.12122096e-01  -7.83707127e-02   0.00000000e+00
  -2.66888547e+00   1.46758423e+01  -1.42084014e+00   1.00000000e+00
   9.20000002e-02   9.81999993e-01   0.00000000e+00   8.99999976e-01
   0.00000000e+00   0.00000000e+00   0.00000000e+00   1.12185276e+00]
We see that the type of the .gs_real array is a Numpy array, and we can print the array to see that it contains 28 floating point numbers with all kinds of viewing information used by O. We can extract the orientation matrix: 
>>> mat = gsreal[4:20]
>>> mat44 = mat.reshape(4,4)
>>> print mat44
[[  0.06268056  -0.05414657   0.30729732   0.        ]
 [ -0.12159095  -0.29289868  -0.02680769   0.        ]
 [  0.28736693  -0.1121221   -0.07837071   0.        ]
 [ -2.66888547  14.67584229  -1.42084014   1.        ]]
Now 'mat' contains the OpenGL 4x4 orientation matrix in column major format. For most uses, the orientation part (upper 3x3) needs to be transposed: 
>>> mat33 = mat44[0:2,0:2]
>>> ori = mat33.transpose()
>>> print ori
[[ 0.06268056 -0.12159095  0.28736693]
 [-0.05414657 -0.29289868 -0.1121221 ]
 [ 0.30729732 -0.02680769 -0.07837071]]
For the translation, we just need to extract that from gsreal: 
>>> trans = gsreal[16:19]
>>> print trans
[ -2.66888547  14.67584229  -1.42084014]

The last bit of information can be found in the _residue_pointers datablock. We also need to format the line correctly, which is straighforward Python. 
>>> resnam= db["alpha_residue_name"]

>>> rp = db["alpha_residue_pointers"]

Download and installation

You can download the OdbParser module HERE or you can get an RPM from MOKs RPMs (xray component). If you choose to compile it yourself, you need to unpack the .tar.gz, move into the directory and go 
python setup.py build
python setup.py install
Of course you need to be root to install into /usr/lib/.

If you subscribe to MOKs RPMs, you simply need to do: 

apt-get install python-odbparser
or, if you prefer YUM: 
yum install python-odbparser
